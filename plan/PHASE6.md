## ① 一句话目标

让评委在 90 秒里记住一帧：**六个无名色块当面吵成你的内心议会，最后所有声音同时安静下来、把决定权郑重交还给你那一秒**——视觉爆点服务于"AI 不拍板、你拍板"的内核，而不是打架分胜负。

---

## ⚠️ 三个被低估的结构性前提（评审 greenlit 的修订基础）

这三点不解决，再多动效都是叠在死等和触顶之上的点缀。**本定稿的"今天必做"档已据此重排。**

1. **debate 在后端全阻塞、零流式（最大翻车点）**：`debate_stream`(213) 对 4 个发言者逐个 `await _debate_one`(187)，**没有 delta 事件**，每条只在整段 LLM 返回后才吐。计算师 `timeout=22s`(200)、每条 flash ~4s，4 条串行正常 ~16-25s；任一条命中 429 还要 `sleep(6)` backoff(205) 再重试。原计划 Beat-2 给的 45s 几乎全被 LLM 等待吃掉——小人跑到位后会**僵在原地干等十几秒**才出下一气泡，arena 跑位(0.7s)/出击(0.5s)动效根本填不满这段死等。**解法不是靠动效掩盖，而是 #1 回放模式 + #3 等待态动画双保险。**
2. **`?demo=1` 跳地牢不省任何 RPM**：会议照样打满 duel2+debate4+verdict1=**7 次 flash**。它只是"操作便利"，**不是延迟/RPM 兜底**。真正的硬兜底只有**预生成 transcript 本地回放**（#1）。
3. **arenaAttack 现有实现已越界内核**：前冲 + 对面 `hit` 闪白(`brightness 2.4`，192) + 头顶劈雷(`arenaLightning`，497) 在讲"攻击-击中-伤害"，与"不分胜负、不消灭任何声音"相悖。**必须去格斗化重构**（A6），而不是再叠 VS/震屏强化它。

## ② Wow 套餐（三档）

### 【第一档 · 今天必做 · 评审 greenlit 的低风险高 wow 集合】

按落地顺序排列（先做 → 后做）。全为纯前端 / 零新增 LLM 调用，任一挂掉自动退回现有纯气泡（`arenaOn=false` 全局兜底）。

- [x] **R. 预生成回放模式 `?replay=1`（最高优先 · 端到端唯一硬兜底）**
  - 做什么：把一场跑顺的 `duel`+`debate`+`verdict` 结果固化成前端 JSON（含每人 `persona/stance/text/query` + verdict 全文），`?replay=1` 时**纯前端按时间轴回放**并驱动全套现有 arena 动效，**零 LLM、零 RPM、零延迟**。天然规避 debate 阻塞死等。
  - 为什么必做：比赛日网络无论如何抖动都能演；它是所有 wow 的承载容器，也是 451/429/超时的终极兜底。`?demo=1` 跳地牢救不了 RPM，只有它能。
  - 落到哪：新增 `state.replay`（一份预录 JSON 常量，或 `static/replay.json`）；`goMeeting`(578) 检测 `?replay=1` 时不发 `/api/meeting/*`，改为本地定时器按节奏依次调用现有 `arenaSpeak`/`addSpeech`/`arenaJoinCamp`/`arenaAttack`(去格斗化后)/`arenaVerdict`/`arenaSit`。后端零改动。
  - 验证：`?replay=1` 全程不发任何 `/api/meeting/*` 请求（Network 面板可证）；断网下整场可走完到建议书。
  - 工作量：**M**（一份预录 JSON + 一个本地时间轴驱动器；约 40 行）。
  - 兜底：回放 JSON 写死永不空；驱动器挂了仍可手动走真实 LLM 路径。**这是今天第一件该做的。**

- [x] **A. 摘下面具的那一秒（无名色块 → 具名队长）** — 评委 #1（均分 88，3/3，top1 真 wow）
  - 做什么：开场六小人以"蒙面/灰度"态登场（不显人格色、不显名字），第一幕队长就位同帧 `reveal-sweep` 揭面亮出人格色光晕 + 名牌；第二幕配角到位时各自逐个揭面。
  - 为什么 wow：把"投射测验"内核做成可见仪式——先看无名内心碎片，再一个个认出"这是我的野心、这是我的恐惧"。全场唯一不可替代的 wow，资源向它倾斜。
  - 落到哪：`makeFighter`(506) 默认加 `.masked`（新增 CSS：`filter:grayscale(1) brightness(.5)`，盖住 `--c` 光晕，名牌 `opacity:0`），并加 `<span class="nameplate">`（默认 hidden）；新增 `arenaReveal(pid)` 去 `.masked` + 播 `.reveal-sweep` 横扫高光 + 显 `pmeta(pid).n`；`arenaSendCaptain`(531) 末尾调 `arenaReveal`；`arenaJoinCamp`(542) 到位 `setTimeout` 后调 `arenaReveal`。
  - 验证：列队时 6 人全灰无名；队长就位帧亮色 + 出名牌；配角到位逐个揭面。
  - 工作量：**M**（约 30 行）。
  - 兜底：① `.masked` 显脏 → 一行把 `grayscale(1)` 退到 `opacity:.6` 剪影态（绝不脏）；② **多层 filter 掉帧风险**（`arenaLineup` 的 `.in` 淡入 + `grayscale` + 揭面 `drop-shadow` 光晕在低端投屏可能打架）→ 揭面退化为简单淡入即成立，且 reveal 与 `.in` 时序若冲突，统一在 `.in` 完成后再触发 `arenaReveal`；③ 图挂 → `arenaOn=false` 退纯气泡。**解说兜一句**："现在还看不清谁是谁——这正是重点，它们都还只是你脑子里没具名的声音。"

- [x] **W. debate 等待态动画（直接救最大翻车点 · 与 R/A 配套）**
  - 做什么：在"已请求 debate 但未返回"的人格上加"起身踱步 / 思考辉光"纯前端动画，把 4 条串行 LLM 的十几秒死等空窗填成"他正在斟酌"，而不是画面冻住。

  - 落到哪：`goMeeting`(578) 中 debate SSE 每收到一人前，对下一位待发言 `pid` 触发 `arenaThinking(pid)`（新增：低频呼吸辉光 + 轻微原地踱步），该人 `text` 到达进 `addSpeech`(439) 时清除。
  - 验证：手动卡慢网络/真实路径下，等待期小人有"斟酌"动画而非僵立。
  - 工作量：**S–M**（一个 `arenaThinking` 态 + keyframes；约 20 行）。
  - 兜底：纯前端，挂了仅退回"僵立"现状，不影响主流程；reduced-motion 下退为静态辉光。

- [x] **B. 拍板时刻 = 全场静止特写 + 半动态金句（Verdict Freeze-Frame）** — 评委 #2/#4/#7/#8 **合并为同一帧**（本场记忆锤）

  - 为什么 wow：top2(3/3)+top4(2/3)+top7/top8 同一帧，所有声音同时安静、把权力交还用户。庄重、克制、不 KO。
  - **金句半动态化（关键改法，零 LLM）**：不要放之四海的标语。用 `verdictPick`(335) + `duelists` 两位名字 + `topic` 前端拼一句**回指用户真实裁决**的话。固定议题"裸辞"下示例：「六个声音吵了一路，没有一个敢替你签那封辞职信。决定，从来都是你的。」`synthesis`（我要综合）分支文案另写：「你选择了综合——所以没有一个声音被淘汰。它们都还在桌上。」

  - 工作量：**S–M**（改现有两个函数 + 一段 keyframes；约 25 行）。
  - 兜底：reduced-motion 下侧移/高光关闭（沿用 205 行规则）；挂了退回静态气泡。

- [x] **C. 固定议题「裸辞」+ 演示导演条 + 限流闸** — 评委 #3（均分 83，3/3）

  - 兜底：它本身是操作兜底；真延迟兜底走 R；最坏全程录屏。

## ④ 验收红线与风险兜底

**红线（逐条勾验）**：
- [ ] **首条插话 < 2s**：本计划全部 wow 零新增 LLM，不碰 `/api/interject`（在地牢）。**保留地牢一道门展示秒级插话**，别因 `?demo=1`/`?replay=1` 跳地牢而放弃这个本来很稳的 wow。
