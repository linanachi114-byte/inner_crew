# PHASE 5 · 2.0「像素小人对战」会议场

> 来源:用户 2.0 构想。把决策会议三幕做成一个**像素小人对战**的可视化：屏幕上方是小人战斗舞台，下方是现有会话气泡。
> 角色立绘提示词见 `docs/persona-sprite-prompts.md`；素材存 `static/sprites/<id>.png`。
> 定位铁律:这是**会议的可视化层**，不是格斗游戏——血条 = 阵营**声势/说服力**（非"命"），**没有 KO / 没人被打死**，最终仍由用户点按钮裁决。炫特效服务于"把内心的会外化看见"，绝不暗示"决定是打出来的"。

## 编排总览（对上后端现有 SSE 事件，零后端改动）

现有事件（`/api/meeting/*`，PHASE2 已实现，不动）：
- **duel** `start{persona,name}`（两队长串行）→ `delta{...}`（流式）→ `end{stance:a|b,text}` → `done{duelists:[a,b]}`
- **debate** 每人 `{persona,name,stance:a|b|third,text,query?}` → `done`
- **verdict** 用户裁决 → JSON 建议书

舞台动作映射：
| 阶段 | 后端事件 | 小人动作 | 声势条 |
|---|---|---|---|
| 开场 | 进入会议 | 六小人居中**列队**淡入 | 两条满格待命 |
| 第一幕 | duel `start` A / B | 队长 A**走向左侧**、B**走向右侧**，转身朝中央 | — |
| 第一幕 | duel `delta` | 发言中队长"说话"抖动/光标，气泡在下方流式 | — |
| 第二幕 | debate 每人 | 该小人从中排**跑向 stance 侧**（a左/b右/third居中），到位后向对面**出击特效** | 对面声势**下挫**、本方**上扬**（封顶不归零） |
| 第三幕 | verdict | 选中侧**肯定姿势**，全员**落座**（呼应"坐下来开会"），进报告 | 定格 |

工程原则:**纯前端表现层**——流程/算分/排序仍在后端纯函数，小人层只"读事件、演动画"（守 CLAUDE.md 架构铁律）。整层是**渐进增强**：sprite 加载失败或一键关闭时，退回现有纯气泡会议，不破坏可用流程与"录屏兜底"。

## 步骤（每步独立可验证；验证通过把 `- [ ]` 改 `- [x]`）

### 1. 素材就位
- 目标:六张 idle sprite 生成入库。产出:`static/sprites/{ambition,guardian,empath,selfcore,logician,dreamer}.png`（透明底、同风格）。
  - [x] 六张文件齐全，浏览器直接访问 `/sprites/ambition.png` 等返回 200、透明底、画风一致。

### 2. 舞台骨架（上下分栏）
- 目标:会议屏 `#s-meeting` 拆成上方 `#arena`（战斗舞台）+ 下方现有 `#speeches`（气泡）。产出:`static/index.html` DOM + CSS。
  - [x] 会议屏上半显示空舞台容器、下半是气泡区；竖屏不挤、可各自滚动；arena 隐藏时退回原布局。

### 3. Sprite 组件 + 人格映射 + 像素渲染
- 目标:`spriteFor(pid)` 按 `persona id → /sprites/{id}.png` 建小人元素；`image-rendering:pixelated`；左侧阵营用 `transform:scaleX(-1)` 翻转朝向。
  - [x] 六个小人能渲染、放大不糊、可水平翻转、底色描边与人格专属色协调。

### 4. 开场列队
- 目标:进入会议时六小人在 arena 居中**一排**淡入（用全部 PERSONAS，不依赖谁是队长）。
  - [x] `goMeeting` 一进会议，六小人居中排开淡入登场。

### 5. 队长走位（第一幕）
- 目标:duel `start` 串行到达——首位判为 A 走左、次位 B 走右（与 PHASE 已有"首位 A/次位 B"约定一致），`transition` 平滑移动并转身朝中央。
  - [x] 两位队长分别滑到左右两侧站定、面朝中央；`done` 的 `duelists` 与走位一致。

### 6. 队长陈述联动
- 目标:duel `delta` 流式时，发言队长播放"说话"动效（轻抖/口型/光标），气泡仍在下方流式（已有，不改气泡逻辑）。
  - [x] 队长发言时其 sprite 有说话动效、停笔即止；下方气泡同步蹦字不受影响。

### 7. 其余选边跑位（第二幕）
- 目标:debate 每事件 → 该小人从中排**跑向 stance 侧**（a 左 / b 右 / third 居中留守），跑动动效 + 到位站入该阵营队列。
  - [x] 4 个配角依次跑向正确阵营；third 留在中间；与下方"靠边站"气泡方向一致。

### 8. 反驳攻击特效
- 目标:配角到位后向**对面阵营**出击：前冲位移 + 命中闪光 + 对面受击抖动（纯 CSS/transform，单张 idle 即可；有 attack 帧则切帧）。
  - [x] 每次反驳都有一次朝对面的攻击特效 + 对面受击反馈；特效不阻塞后续事件。

### 9. 声势条（复用阵营天平）
- 目标:arena 两侧各一条**声势/说服力条**，数据复用 `tally_camps`/现有 `updateBalance` 的 a∶b 计票；攻击时对面下挫、本方上扬，**有地板、不归零**（杜绝 KO 观感）。
  - [x] 选边推进时两条声势条此消彼长、始终 >0；与天平读数一致。

### 10. 裁决收束（第三幕）
- 目标:用户裁决后，选中侧小人播放**肯定姿势**（非击杀），随后**全员落座**呼应"把这场没开成的会吵完/坐下来"，再进报告。
  - [x] 裁决后有收束动画、无 KO/死亡观感、顺滑进入建议书。

### 11. 降级兜底 + 性能 + 移动端
- 目标:① sprite 任一加载失败或开关关闭 → 自动退回纯气泡会议（铁律:不破坏可用流程/录屏兜底）；② 动画只用 `transform/opacity`；③ 尊重 `prefers-reduced-motion`；④ 竖屏布局可用。
  - [x] 断网/删图测试仍能完整开完会（纯气泡）；手机竖屏可玩；reduced-motion 下退化为淡入淡出。

### 12. 验收
- 目标:端到端实测一场完整对战会议 + 截图。
  - [x] 一场会议:列队→队长分边→配角跑位反驳带特效→声势条联动→裁决收束，端到端**不突破现有 ≤90s 红线**；截图存 `docs/screenshots/`。

## 砍单优先级（2.0 时间不够按序砍）

攻击帧切换（退回单张 idle + CSS 特效）→ 说话动效（退回静态）→ 跑位过渡动画（退回瞬移到位）→ 声势条动画（退回现有天平条）→ 整个像素层（退回 1.0 纯气泡会议）。
**永不砍**:① 纯气泡会议始终可用（降级兜底）；② 不引入"KO/分胜负"观感；③ 不改后端事件契约。

## 验收红线
- 像素层是**纯前端 + 渐进增强**：关掉它，1.0 流程一字不差地跑。
- 端到端仍 ≤90s；Demo 3 分钟。
- 全程无"小人被打死/血条归零"，裁决权始终在用户。
